# ◉ Why AI is used

⟦ ***Concept*** ⟧

For me, IdealOS is primarly a learning project.

I don't only want to setup an operating system, but also learning about OS design, µKernels, MM, security models, runtimes, and all the cursed things hidden behind them.

As a HS student with a limited amount of free time due to a lot of things which I will be very pleased to detail you on Discord, I actively use AI tools during development.

◆ What I use AI for

• Learning unfamiliar concepts
• Reviewing code
• Generating prototypes
• Exploring alternative implementations
• Identifying bugs
• Improving documentation
• Discussing architectural decisions

# ◉ Current State of the Codebase

⟦ ***Reality Check*** ⟧

Because I'm against AI as core conceptor and realizer of the project, this current codebase SHOULD NOT be considered as the final implementation.

Actually, ideas are mine, and original code also, but I used AI reviews on EVERY line and many of my original writings were deleted by AI.

Some modules were originally written manually and later heavily refactored with AI assistance.

Other components may have been generated from prompts and then modified by hand.

◆ What this means

✓ This code represents what I currently want to build.

✓ This code does not necessarily represent how I would build it in a mature version of the project.

✓ This code is a fast implementation.

✓ This code is not the final implementation.

# ◉ Long-Term Goal

⟦ ***Future*** ⟧

As the project matures, many subsystems will likely be redesigned, rewritten, replaced, broken, rebuilt, broken again, and hopefully improved.

The objective is to progressively build a coherent and robust architecture while improving my understanding of the underlying concepts.

The current implementation is mainly a vehicle for experimentation.

Once the fast implementation of the µKernel is finished, I plan to rewrite basically everything with a much better understanding of what I'm doing.

Or at least with a slightly less terrible understanding.
